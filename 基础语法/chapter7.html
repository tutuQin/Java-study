第七章 面向对象(Object-Oriented Programming)

面向对象编程(OOP)是 Java 的核心范式。Java 把世界中的"事物"抽象为类(Class),把每个具体的事物表示为对象(Object),通过封装、继承、多态三大特性组织代码,让程序结构更清晰、复用性更强。

7.1 面向对象概述

① 面向过程 vs 面向对象

对比项 面向过程(POP) 面向对象(OOP)
思考方式 "怎么做" —— 步骤、函数 "谁来做" —— 对象、职责
基本单位 函数 + 全局数据 类 + 对象(数据与行为绑定)
典型语言 C、Pascal Java、C++、Python、C#
优势 简单直接,性能好 可维护、可扩展、可复用
劣势 难以应对复杂业务 设计成本高,初学者门槛高 
// 面向过程:洗衣服
washClothes(clothes);   // 步骤: 放水→放洗衣粉→搅拌→脱水...

// 面向对象:让洗衣机自己洗
WashingMachine m = new WashingMachine();
m.wash(clothes);        // 我们不关心内部细节

② OOP 三大特性

特性 关键字/手段 核心思想
封装(Encapsulation) private + getter/setter 把数据与操作打包,隐藏内部实现细节
继承(Inheritance) extends 子类复用父类代码,体现"is-a"关系
多态(Polymorphism) 父类引用 + 方法重写 同一方法在不同对象上有不同行为
💡 抽象(Abstraction):很多教材会把"抽象"作为第四大特性。抽象是把现实事物简化为类的过程,是 OOP 的基础思维方式。

③ 类与对象的关系

  • 类(Class):是一种"模板"或"蓝图",描述对象的属性和行为,本身不占内存。
  • 对象(Object):是类的一个具体"实例",根据类的定义创建,占用堆内存。
  • 一个类可以创建任意多个对象,每个对象都有独立的属性副本。
// 类是图纸,对象是按图纸造出来的房子
// Student 类  →  zhangSan(对象)、liSi(对象)、wangWu(对象) ...

7.2 类与对象

① 类的定义

// 语法
[修饰符] class 类名 {
    // 成员变量(属性)
    // 构造方法
    // 成员方法(行为)
    // 代码块、内部类...
}

// 示例:学生类
public class Student {
    // 成员变量
    String name;
    int age;

    // 成员方法
    public void study() {
        System.out.println(name + " 正在学习");
    }
}
📌 命名约定:类名使用大驼峰(UpperCamelCase),如 UserService、OrderItem;每个 .java 文件中只能有一个 public 类,且文件名必须与该类名相同。

② 对象的创建与使用

// 创建对象:new + 构造方法
Student s = new Student();

// 访问成员变量
s.name = "张三";
s.age  = 18;

// 调用成员方法
s.study();   // 输出:张三 正在学习

// 一个类可以创建多个对象,各自独立
Student a = new Student();
Student b = new Student();
a.name = "A";
b.name = "B";
System.out.println(a.name);   // A
System.out.println(b.name);   // B

③ 对象在内存中的结构

栈(Stack)                堆(Heap)
┌──────────┐            ┌──────────────────┐
│ s ──────────────────→ │ Student 对象       │
│ (引用,4 字节)         │ ┌──────────────┐ │
│           │           │ │ name = "张三"│ │
│           │           │ │ age  = 18    │ │
└──────────┘            │ └──────────────┘ │
                        └──────────────────┘
  • 栈:存放局部变量(包括对象引用)
  • 堆:存放 new 出来的对象本身
  • 方法区/元空间:存放类的元信息、静态变量、字符串常量池

④ 匿名对象

没有名字、用完即弃的对象,适合一次性调用。

new Student().study();    // 创建后立刻调用 study,再无引用

// 常用于做参数
print(new Student());
⚠️ 注意:匿名对象使用完后会成为垃圾等待 GC 回收,不能再次访问。多次使用同一对象应保留引用。

7.3 成员变量与方法

① 成员变量(字段 / Field)

定义在类中、方法外的变量。每个对象都有自己独立的一份。

public class Person {
    String name;          // 默认值 null
    int age;               // 默认值 0
    boolean married;       // 默认值 false
}

成员变量 vs 局部变量

对比项 成员变量 局部变量
位置 类中、方法外 方法内、代码块内、形参
默认值 有(0、null、false) 无,必须手动初始化
作用域 整个类 所属代码块
存储位置 堆(随对象) 栈(随方法)
生命周期 对象创建到销毁 方法调用到方法返回

② 成员方法

定义在类中的方法,代表对象的行为。语法和"普通方法"一致,但必须通过对象调用(非 static 时)。

public class Calculator {
    private int result;

    public void add(int x) {
        result += x;
    }

    public int getResult() {
        return result;
    }
}

Calculator c = new Calculator();
c.add(5);
c.add(3);
System.out.println(c.getResult());   // 8
📌 提示:方法的详细语法(参数、返回值、重载、可变参数)见第六章 方法。本章重点关注方法在 OOP 中的作用。

7.4 构造方法(Constructor)

构造方法用于创建对象时的初始化,在 new 时自动调用。

① 定义规则

  • 方法名必须与类名完全一致
  • 没有返回值类型(连 void 都不能写)
  • 可以有参数,可以重载
  • 访问修饰符通常是 public(也可以是 private,如单例模式)
public class Student {
    String name;
    int age;

    // ① 无参构造
    public Student() {
        System.out.println("无参构造被调用");
    }

    // ② 有参构造(重载)
    public Student(String name, int age) {
        this.name = name;
        this.age  = age;
    }
}

Student s1 = new Student();              // 调用 ①
Student s2 = new Student("张三", 18);     // 调用 ②

② 默认构造方法

如果一个类没有定义任何构造方法,编译器会自动生成一个无参构造(方法体为空)。一旦定义了任意构造方法,默认构造就不再生成。 

public class A { }                // 自动生成 public A() { }

public class B {
    public B(int x) { }       // 定义了有参构造
}
// new B();      ❌ 编译错误,没有无参构造
// new B(10);    ✅
💡 最佳实践:定义有参构造时,通常顺手再写一个无参构造,以便框架(如 Spring、Hibernate)通过反射创建对象。

③ this() 调用其他构造方法

public class Student {
    String name;
    int age;

    public Student() {
        this("无名", 0);          // 调用另一个构造方法
    }

    public Student(String name, int age) {
        this.name = name;
        this.age  = age;
    }
}
⚠️ 限制:this(...) 必须在构造方法的第一行,且不能形成循环调用。

7.5 this 关键字

this 代表当前对象的引用,即"哪个对象调用这个方法,this 就指向谁"。

① 三大用法

用法 含义
this.成员变量 访问当前对象的成员变量(区分形参与字段重名)
this.方法() 调用当前对象的方法(通常可省略 this)
this(...) 在构造方法中调用本类的其他构造 
public class Box {
    int width, height;

    public Box(int width, int height) {
        this.width  = width;     // this 区分字段与形参
        this.height = height;
    }

    public int area() {
        return width * height;   // 等价于 this.width * this.height
    }

    public Box grow() {
        return new Box(this.width + 1, this.height + 1);
    }
}
📌 注意:this 不能在 static 方法中使用 —— 静态方法属于类,不依赖具体对象。

7.6 封装(Encapsulation)

封装是 OOP 第一大特性,核心思想:隐藏内部实现,只暴露必要接口。具体做法是把字段设为 private,通过 public 的 getter/setter 控制访问。

① 访问修饰符

修饰符 同类 同包 子类 其他包
private ✅ ❌ ❌ ❌
默认(无修饰符) ✅ ✅ ❌ ❌
protected ✅ ✅ ✅ ❌
public ✅ ✅ ✅ ✅

② 封装示例

public class Account {
    private String owner;
    private double balance;    // 私有,外部无法直接修改

    public String getOwner() { return owner; }
    public void setOwner(String owner) { this.owner = owner; }

    public double getBalance() { return balance; }

    // 不直接提供 setBalance,而是用业务方法
    public void deposit(double amount) {
        if (amount <= 0) throw new IllegalArgumentException("金额必须为正");
        balance += amount;
    }

    public void withdraw(double amount) {
        if (amount > balance) throw new IllegalStateException("余额不足");
        balance -= amount;
    }
}

③ 封装的优势

  • 数据安全:防止外部直接修改,可在 setter 中加校验
  • 易维护:内部实现可以随意改,只要接口不变,调用方就不受影响
  • 低耦合:对象之间只通过公开方法交互
  • 规范化:符合 JavaBean 规范,便于框架反射

④ JavaBean 规范

一种约定俗成的"标准类",广泛用于 Spring、MyBatis、Jackson 等框架。

public class User {                 // ① 必须是 public 类
    private String name;            // ② 字段全部 private
    private int age;

    public User() { }                // ③ 提供无参构造

    public String getName() { return name; }  // ④ getter/setter
    public void setName(String name) { this.name = name; }

    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}

7.7 继承(Inheritance)

继承通过 extends 关键字实现:子类可以复用父类的字段和方法,并在此基础上扩展。继承体现了"is-a"关系(狗 is-a 动物)。

① 基本语法

// 父类(基类、超类)
public class Animal {
    String name;
    public void eat() {
        System.out.println(name + " 在吃东西");
    }
}

// 子类
public class Dog extends Animal {
    public void bark() {
        System.out.println(name + " 汪汪叫");
    }
}

// 使用
Dog d = new Dog();
d.name = "旺财";
d.eat();    // 继承自 Animal
d.bark();   // Dog 自有方法

② Java 继承的特点

  • 单继承:一个类只能 extends 一个父类(C++ 支持多继承,Java 不支持以避免菱形问题)
  • 多层继承:A → B → C,C 间接继承 A 的所有可见成员
  • 所有类默认继承自 java.lang.Object
  • 子类无法继承父类的:构造方法、private 成员(可以拥有但访问不到)
  • final 修饰的类不能被继承(如 String、Integer)

③ 继承中的初始化顺序

父类静态字段、静态代码块  →  子类静态字段、静态代码块
父类实例字段、实例代码块  →  父类构造方法
子类实例字段、实例代码块  →  子类构造方法
    
class A {
    public A() { System.out.println("A 构造"); }
}
class B extends A {
    public B() { System.out.println("B 构造"); }
}
new B();
// 输出:
// A 构造
// B 构造
⚠️ 注意:子类构造方法第一行会隐式调用 super()(父类无参构造)。如果父类没有无参构造,子类必须显式调用 super(参数)。

7.8 super 与方法重写

① super 关键字

super 代表父类对象的引用,用于在子类中显式访问父类的成员。

用法 说明
super.字段 访问父类的字段(与子类同名时)
super.方法() 调用父类的方法(子类已重写时也能访问父类版本)
super(...) 调用父类构造,只能放在子类构造方法第一行 
class Animal {
    String name;
    public Animal(String name) { this.name = name; }
    public void sleep() { System.out.println(name + " 睡觉"); }
}

class Cat extends Animal {
    public Cat(String name) {
        super(name);          // 调用父类构造
    }
    public void sleep() {
        super.sleep();        // 先调用父类逻辑
        System.out.println("打呼噜");
    }
}

② 方法重写(Override)

子类重新实现父类的方法,以提供更特化的行为。

class Animal {
    public void sound() { System.out.println("动物叫"); }
}

class Dog extends Animal {
    @Override
    public void sound() {
        System.out.println("汪汪!");
    }
}

重写的规则(三同两小一大)

项目 规则
方法名 必须相同
参数列表 必须相同(包括类型、顺序、个数)
访问修饰符 子类不能更严格(public → public/protected ✅,public → private ❌)
返回值类型 相同或子类(协变返回类型)
抛出异常 子类抛出的检查异常,不能比父类更多/更宽
static / final / private 不能被重写(static 是隐藏,private 子类根本看不到)

重写(Override)vs 重载(Overload)

对比项 重写(Override) 重载(Overload)
发生在 父类与子类之间 同一个类中
方法名 相同 相同
参数 必须相同 必须不同
返回值 相同或子类 可以不同(不能仅靠返回值区分)
多态绑定 运行时(动态绑定) 编译时(静态绑定)
💡 提示:始终为重写方法加 @Override 注解 —— 编译器会检查是否真的"重写"了某个父类方法,防止拼写错误造成"不重写而新增方法"的 bug。

7.9 多态(Polymorphism)

多态是 OOP 第三大特性。同一个方法调用,在不同对象上呈现不同的行为。多态的核心是"运行时动态绑定"。

① 多态的三个必要条件

  • 有继承关系(或实现接口)
  • 子类重写父类方法
  • 父类引用指向子类对象(向上转型)
class Animal { public void sound() { System.out.println("…"); } }
class Dog extends Animal { public void sound() { System.out.println("汪汪"); } }
class Cat extends Animal { public void sound() { System.out.println("喵喵"); } }

Animal a;
a = new Dog(); a.sound();   // 输出:汪汪
a = new Cat(); a.sound();   // 输出:喵喵

② 向上转型与向下转型

// ① 向上转型(子类 → 父类):自动,安全
Animal a = new Dog();      // Dog 被视为 Animal
a.sound();                      // ✅ 多态:调用 Dog 的 sound
// a.bark();                    ❌ Animal 没有 bark 方法

// ② 向下转型(父类 → 子类):必须强转,可能 ClassCastException
if (a instanceof Dog) {
    Dog d = (Dog) a;          // 安全
    d.bark();
}

// JDK 16+ 模式匹配,更简洁
if (a instanceof Dog d) {
    d.bark();
}

③ 多态中的方法访问规则

成员 编译看左边(父类),运行看?
成员变量 编译看左,运行也看左(无多态,字段不重写)
静态方法 编译看左,运行也看左(静态方法是隐藏,不是重写)
实例方法 编译看左,运行看右(子类)—— 真正的多态

④ 多态的优势

// 不用多态:每加一个动物就要写一个方法
public void playDog(Dog d) { d.sound(); }
public void playCat(Cat c) { c.sound(); }

// 使用多态:一个方法搞定所有 Animal
public void play(Animal a) { a.sound(); }

play(new Dog());
play(new Cat());
play(new Bird());   // 新增类型,play 方法不需改动
💡 OCP 开闭原则:多态是实现"对扩展开放,对修改关闭"的关键。新增子类时,无需修改使用父类引用的代码。

⑤ 为什么多态必须依赖继承?——「动物 vs 石头」类比

很多初学者会问:既然只是想让不同对象表现出不同行为,为什么非要继承?直接写一个 shout() 方法在每个类里不行吗?这个问题的答案,藏在 Java 作为强类型语言的底层逻辑里。下面用三条"铁律"来彻底说明。

铁律一:父类是一份「行为规约(契约)」

"动物都会叫"是全天下动物的共性。父类 Animal 里声明一个 shout() 方法,本质上根本不是为了实现"怎么叫",而是确立一份规约。它是对编译器和整个系统发出的官方宣告:

📜 「凡是属于我 Animal 类型的生物,我保证它一定具备『叫』这个行为!」
public abstract class Animal {
    // 这不是实现,这是"宣告":所有 Animal 都必须能 shout
    public abstract void shout();
}

铁律二:继承是子类获取「合法通行证」的唯一方式

狗会"汪汪",猫会"喵喵",这是各自的重写表现(多态)。但它们必须在类名后写上 extends Animal。这行代码的本质,就是去"类型校验处"登记盖章:

public class Dog extends Animal {       // ← 这行就是"登记盖章"
    @Override
    public void shout() { System.out.println("汪汪"); }
}

编译器一看:"哦,你继承了 Animal,那你也属于 Animal 类型了。既然你是 Animal,根据规约,你肚子里的 shout() 方法就合法了,我给你发绿灯通行证。"

铁律三:类型检查是运行前的「安检大闸」

这是最关键的一条。假设我们写一个"让动物叫"的通用方法:

public void makeAnimalShout(Animal am) {
    am.shout();
}
  • 🛡️ 安全边界:这个方法在门口设立了极其严格的安检 —— "只有持有 Animal 通行证的类型才可以进去!"
  • 🚫 拦截错误:此时如果有人误传一个 Stone(石头)进去 —— 因为 Stone 根本没有继承 Animal,它手里没有那张统一的通行证。
  • ⚡ 当场击杀:Java 编译器在编译那一秒(还没运行!),一眼就看到类型不对,当场亮起红灯。
class Stone { }   // 石头没有继承 Animal,也没有 shout 方法

makeAnimalShout(new Dog());     // ✅ Dog extends Animal,放行
makeAnimalShout(new Cat());     // ✅ Cat extends Animal,放行
makeAnimalShout(new Stone());   // ❌ 编译错误!Stone 没通行证,根本进不了门
// error: incompatible types: Stone cannot be converted to Animal
📌 关键洞察:编译器在编译期就把石头挡在门外了,程序还没运行就拒绝通过。这就是强类型语言带来的"零运行时风险"。

🏆 思想闭环

多态的"多态"二字,指的是运行时猫喵喵、狗汪汪的多样表现;

但多态之所以能安全落地的"大前提",全靠父类在编译期用"同一类型"死死卡住的类型检查。

如果不通过继承去定义这个统一的类型,强类型语言就没办法在程序运行前精准判断它到底有没有这个方法。没有这个规约,万能方法就成了空中楼阁。

阶段 谁在工作 做什么
编译期 编译器(静态类型检查) 检查传入对象是否属于 Animal 类型(看通行证)
运行期 JVM(动态分派) 根据对象的真实类型调用对应的 shout() 实现
⚠️ 对比弱类型语言:Python 的"鸭子类型"哲学是"只要它会叫,我就当它是鸭子",不强制继承 —— 灵活但风险高,错误要到运行时才暴露。Java 选择了相反的方向:用编译期类型检查,把错误消灭在运行之前。

7.10 抽象类(Abstract Class)

抽象类是用 abstract 修饰的类,常作为"半成品",规定子类必须实现的方法,自己不能被实例化。

① 抽象类与抽象方法

// 抽象类
public abstract class Shape {
    String color;

    // 普通方法(有方法体)
    public void printColor() {
        System.out.println("颜色:" + color);
    }

    // 抽象方法(无方法体,以 ; 结尾)
    public abstract double area();
}

// 子类必须实现所有抽象方法
public class Circle extends Shape {
    double radius;
    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
}

// Shape s = new Shape();   ❌ 抽象类不能实例化
Shape s = new Circle();           // ✅ 通过子类

② 抽象类的特点

  • 使用 abstract 关键字修饰
  • 不能直接实例化(new Shape() 编译错误)
  • 可以有构造方法,供子类 super() 调用
  • 可以包含抽象方法和普通方法
  • 可以没有抽象方法(但很少见)
  • 子类必须实现所有抽象方法,否则子类也得是抽象类
  • abstract 不能与 final、private、static 共用

③ 适用场景

当多个类有共同的字段和部分共同行为,但某些行为需要由各子类自己实现时,使用抽象类。例如:模板方法模式。

public abstract class AbstractDao {
    // 模板方法:固定流程
    public final void execute() {
        connect();
        doExecute();      // 由子类决定具体怎么执行
        close();
    }
    protected void connect()  { /* 连接数据库 */ }
    protected void close()    { /* 关闭连接 */ }
    protected abstract void doExecute();
}

7.11 接口(Interface)

接口是完全抽象的"协议",定义"能做什么",不关心"怎么做"。Java 中接口是实现多继承(行为)和面向接口编程的关键。

① 定义与实现

// 定义接口:interface 关键字
public interface Flyable {
    // 接口中的方法默认是 public abstract
    void fly();
}

public interface Swimmable {
    void swim();
}

// 类实现接口:implements 关键字,可实现多个
public class Duck implements Flyable, Swimmable {
    @Override
    public void fly()  { System.out.println("鸭子飞"); }
    @Override
    public void swim() { System.out.println("鸭子游"); }
}

② 接口的成员特点

成员 默认修饰符 说明
字段 public static final 必须显式初始化,是常量
抽象方法 public abstract 默认就是抽象的
默认方法(JDK 8+) public default 有方法体,供实现类继承或重写
静态方法(JDK 8+) public static 通过接口名调用
私有方法(JDK 9+) private 仅供接口内部其他默认/静态方法复用 
public interface Vehicle {
    int MAX_SPEED = 300;        // 等价于 public static final int MAX_SPEED = 300;

    void run();                  // 抽象方法

    default void brake() {       // 默认方法,实现类可直接用
        log("刹车");
    }

    static void info() {         // 静态方法
        System.out.println("我是 Vehicle 接口");
    }

    private void log(String msg) { // 私有方法 JDK 9+
        System.out.println("[LOG] " + msg);
    }
}

Vehicle.info();                 // 静态方法直接通过接口名调

③ 接口的继承

接口之间可以多继承(extends),一个接口能继承多个父接口。

public interface A { void a(); }
public interface B { void b(); }

public interface C extends A, B {
    void c();
}

// 实现 C 必须实现 a() / b() / c() 三个方法

④ 接口 vs 抽象类

对比项 接口(interface) 抽象类(abstract class)
继承 类可实现多个接口 类只能继承一个抽象类
字段 只能是 public static final 常量 任意字段
构造方法 没有 有
方法 抽象/默认/静态/私有方法 普通/抽象方法
定位 "能做什么" —— 行为契约 "是什么" —— 模板/部分实现
关键字 implements extends
💡 选择原则:能用接口就用接口,需要共享代码或保留状态时才用抽象类。两者也常配合使用 —— 抽象类实现接口,处理共性,子类专注差异。

⑤ 函数式接口

只有一个抽象方法的接口,可以用 Lambda 表达式实现,常加 @FunctionalInterface 注解(让编译器检查)。

@FunctionalInterface
public interface Calculator {
    int calc(int a, int b);
}

Calculator add = (a, b) -> a + b;
System.out.println(add.calc(3, 5));   // 8

7.12 static 关键字

static 表示"属于类,而不是属于对象"。所有对象共享同一份,无需创建对象即可使用。

① 静态变量(类变量)

public class Student {
    private static int count = 0;     // 所有 Student 共享
    private String name;                 // 实例变量,每个对象一份

    public Student(String name) {
        this.name = name;
        count++;                          // 每创建一个学生,总数 +1
    }
    public static int getCount() { return count; }
}

new Student("A");
new Student("B");
System.out.println(Student.getCount());   // 2

② 静态方法

  • 不依赖具体对象,可以直接用类名调用(推荐),也可用对象调用(不推荐)
  • 不能使用 this、super
  • 不能直接访问实例成员(可访问其他静态成员)
  • 常用于工具方法,如 Math.max()、Integer.parseInt()
public class MathUtil {
    public static int square(int x) {
        return x * x;
    }
}
MathUtil.square(5);     // 类名直接调用

③ 静态代码块

类加载时执行一次,常用于初始化静态资源(如读取配置、注册驱动)。

public class Config {
    static Properties props;
    static {
        props = new Properties();
        // 加载配置文件
        props.load(Config.class.getResourceAsStream("/app.properties"));
    }
}

四种代码块的执行顺序

class Demo {
    static { /* ① 静态代码块,类加载执行 1 次 */ }
    {        /* ② 实例代码块,每次 new 执行   */ }
    public Demo() { /* ③ 构造方法            */ }
}
// 执行顺序:① → (每次 new 时) ② → ③

④ 静态导入(JDK 5+)

import static java.lang.Math.*;

double r = sqrt(16);     // 不再需要 Math.sqrt
double p = PI;
⚠️ 注意:静态导入虽然简洁,但会降低代码可读性,过度使用反而让人弄不清符号出处,慎用。

⑤ main 方法为什么是 static?

JVM 启动时还没有任何对象,无法通过对象调用 main,所以必须用 static 让 JVM 通过类名直接调用。

public static void main(String[] args) { ... }

7.13 final 关键字(进阶)

第 2.4 节已介绍 final 的基本用法,这里补充 OOP 相关的特性。

修饰 效果 典型示例
变量 常量,只能赋值一次 final int MAX = 100;
方法 禁止子类重写 Object.getClass()
类 禁止继承,所有方法隐式 final String、Integer、Math
形参 方法内不能再赋值,Lambda 必须捕获 final/有效 final void m(final int x) 
// final 类无法继承
public final class Constants { ... }
// class MyConstants extends Constants { } ❌ 编译错误

// final 方法无法重写
public class Base {
    public final void id() { ... }
}
public class Sub extends Base {
    // public void id() { } ❌ cannot override
}
💡 不变性(Immutability):所有字段 private final + 不提供 setter + 类用 final 修饰,即可造一个不可变类。不可变对象天然线程安全,如 String、BigDecimal。

7.14 内部类(Inner Class)

定义在一个类内部的类,可以访问外部类的私有成员,常用于封装辅助逻辑。

① 四种内部类

类型 位置 能否访问外部 this 用途
成员内部类 类中、方法外 ✅ 与外部类紧密关联的辅助类
静态内部类 类中,加 static ❌ 不依赖外部实例的工具/Builder
局部内部类 方法内 ✅ 极少用
匿名内部类 表达式中 ✅ 事件回调、接口的快速实现

② 成员内部类

public class Outer {
    private String msg = "hi";

    public class Inner {
        public void show() {
            System.out.println(msg);   // 可直接访问外部私有字段
        }
    }
}

// 创建必须先有外部对象
Outer o = new Outer();
Outer.Inner i = o.new Inner();
i.show();

③ 静态内部类

public class Outer {
    private static int count;
    public static class Builder {
        public Outer build() { count++; return new Outer(); }
    }
}
Outer o = new Outer.Builder().build();     // 无需外部对象

④ 匿名内部类

在创建对象的同时定义类,适用于只用一次的实现。

// 实现接口
Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("线程跑起来");
    }
};
new Thread(r).start();

// JDK 8 后函数式接口可用 Lambda 替代,更简洁
new Thread(() -> System.out.println("线程跑起来")).start();
📌 注意:匿名内部类访问外部局部变量时,该变量必须是 final 或事实上的 final(JDK 8+ 自动识别,只要没被修改过)。

7.15 包(Package)与访问控制

① 包的作用

  • 组织类:把功能相关的类放在同一个包
  • 避免命名冲突:不同包可以有同名类
  • 访问控制:配合 default / protected 限定可见范围
// 文件 src/com/example/user/User.java 第一行
package com.example.user;

// 使用其他包的类需 import(java.lang 包默认导入)
import java.util.List;
import java.util.ArrayList;
import java.util.*;        // 通配符导入

public class UserService { ... }

② 包命名规范

  • 全部小写字母
  • 反转的公司域名开头,如 com.alibaba.fastjson
  • 层次结构:公司.项目.模块.功能,如 com.example.order.service

③ 四种访问级别(重温)

修饰符 同类 同包 子类 其他包 典型用法
private ✅ ❌ ❌ ❌ 字段、辅助方法
默认 ✅ ✅ ❌ ❌ 包内共享
protected ✅ ✅ ✅ ❌ 给子类的扩展点
public ✅ ✅ ✅ ✅ 对外 API
💡 最小可见性原则:能用 private 就别用 default,能用 default 就别用 public。把"暴露给外界的接口"控制在最小集合,降低耦合。

④ 用"社区安保"来理解四种权限

把 Java 的访问权限（Access Modifiers）想象成一个大社区的安全安保级别。在社区里有四个核心概念，从最亲密到最陌生排列：

  1. 自己家（同一个类 / Class）：只有你自己能碰的东西。
  2. 同一个小区（同一个包 / Package）：邻居们抬头不见低头见，有些东西可以共享。
  3. 亲戚家（子类 / Subclass）：不管亲戚住得远还是近，血缘关系在这，能继承家产。
  4. 大马路（外部所有人 / World）：路过的陌生人。

④-a private（私有的）—— 你的银行卡密码

  • 安全级别：最高（最封闭）
  • 通俗解释：这是你自己家（类）绝对私密的东西，除了你本人，谁也别想碰。
  • 生活场景：你的银行卡密码、日记本。哪怕是住你隔壁的邻居（同包），或者你亲生儿子（子类），都不能直接看。
  • 代码表现：只能在定义它的那个类内部使用。

④-b default（默认 / 包访问权限）—— 放在客厅的电视机

  • 安全级别：次高（注意：在代码里不需要写任何关键字，不写就是它）
  • 通俗解释：这是属于同一个小区（同一个包 Package）的共享物资。只要是小区的居民，谁都能来用；但只要出了小区大门，外人一概不给碰。
  • 生活场景：你家客厅的电视。你爸妈、你兄弟姐妹（同包的类）能看。但如果你的亲生儿子长大了搬到了别的小区（跨包的子类），他也不能天天跨区回你家客厅看电视。
  • 代码表现：只有在同一个包（Package）底下的类才能访问。

④-c protected（受保护的）—— 你的传家宝

  • 安全级别：较低（开始搞特殊化了）
  • 通俗解释：这个级别最讲究"血缘和交情"。同一个小区（同包）的邻居可以碰；搬到外地去的后代子孙（子类）也可以碰。
  • 生活场景：你们家的传家宝。同一个屋檐下的家人（同包）能看，哪怕你儿子以后出国定居了（跨包的子类），因为他有继承权，他依然可以继承和使用这个传家宝。但没有任何关系的纯路人是拿不到的。
  • 代码表现：同包的类，以及任何地方的子类都能访问。

④-d public（公共的）—— 摆在大街上的长椅

  • 安全级别：最低（完全公开）
  • 通俗解释：太平洋警察，谁都能管。不管是不是一个类的、是不是一个包的、有没有继承关系，通通放行。
  • 生活场景：公园的长椅、路边的红绿灯。谁路过谁都能用。
  • 代码表现：任何地方的任何类都能访问。
💡 为什么需要这些权限？
想象一下，如果 Java 里没有这些限制，所有人都是 public：你写了一个银行系统，里面有一个变量叫 double money（余额）。因为是完全公开的，隔壁班小明写的新程序随手就能改成 money = 0;。你哭都没地方哭。

所以，Java 的设计思想是："该藏的藏起来（private），该暴露的才暴露（public）。"这就叫封装。我们通常把属性（比如年龄、余额）藏成 private，然后留一个 public 的方法（比如 changeAge()）让你通过正规渠道去修改。这样，你既保护了数据的安全，又给了别人沟通的接口。

7.16 单例模式(Singleton Pattern)

既然聊完了访问权限，那讲单例模式（Singleton Pattern）就太顺理成章了！因为单例模式的核心秘密，恰恰就藏在我们刚刚讲到的 private（私有）权限里。

用通俗的大白话来说：单例模式，就是保证整个系统里，某个类"有且仅有一个实例（对象）"的设计方法。

① 为什么要搞"单例"？

  • 古代的皇帝：一个朝代在同一时刻，只能有一个皇帝。如果同时出现两个皇帝，天下大乱。
  • 电脑的回收站：你在电脑里打开回收站，不管双击多少次，弹出来的都是同一个回收站窗口，它不会给你新建一堆回收站。

在程序里也是一样，像数据库连接池、线程池、网站的计数器、配置文件读取器。这些东西如果满天飞，不仅极其浪费内存，还会导致数据错乱（比如两个计数器同时记不同的数）。所以，我们必须保证它全国限购一辆，绝无分店。

② 怎么用 Java 代码卡死"只能有一个"？

如果让你来设计，你怎么防止别人在外面不断地 new 出来新对象呢？答案：把构造函数私有化（用 private）。

这就是单例模式的核心三步走：

  1. 藏住构造鞋厂：不让外面 new。把构造方法改成 private，这样外面的类就没办法通过 new 那个类() 来创建对象了。
  2. 自己内部生一个：既然外面不能 new，那就自己在类里面 new 好一个唯一的对象，藏在私有变量里。
  3. 留个后门送出去：提供一个公共的（public）静态方法，当别人想要这个对象时，把刚刚自己内部 new 好的那唯一一个对象送给对方。

③ 饿汉式（迫不及待，先创建再说）

故事：这个人快饿死了，一刻也不能等。程序刚一启动，不管有没有人要用这个对象，他先把对象 new 出来摆在那。

public class SingleEager {
    // 1. 内部自己先 new 好一个，并且用 private static 锁死
    private static final SingleEager instance = new SingleEager();

    // 2. 构造方法私有化！外面的路人没法 new 了
    private SingleEager() {}

    // 3. 留一个公开的接口，别人找你要，你就把唯一的 instance 递出去
    public static SingleEager getInstance() {
        return instance;
    }
}
  • 优点：极其安全，绝对不会出现"不小心创建了两个"的情况（线程安全）。
  • 缺点：占地方。如果这个对象很大，而且从头到尾都没人调用过它，那它一启动就占着内存，属于"占着茅坑不拉屎"。

④ 懒汉式（不见兔子不撒网，用时再创建）

故事：这个人极度重度拖延症。程序启动时他不创建对象。只有当别人第一次过来喊："喂！给我个对象！"的时候，他才慢吞吞地看一眼："哦，还没创建呢，那我这时候再 new 一个吧。" 之后别人再要，他就把之前 new 好的那个给出去。

public class SingleLazy {
    // 1. 先不 new，只写个牌子摆在这，初始是 null
    private static SingleLazy instance = null;

    // 2. 同样构造方法私有化
    private SingleLazy() {}

    // 3. 别人要的时候，再判断要不要 new
    // 注意：这里加了 synchronized 是为了防止多个人同时进来，不小心 new 了两个（线程安全）
    public static synchronized SingleLazy getInstance() {
        if (instance == null) {
            instance = new SingleLazy(); // 只有第一次需要时才 new
        }
        return instance;
    }
}
  • 优点：节约内存，用到了才创建（延迟加载）。
  • 缺点：因为要照顾多线程排队拿对象的问题，加了锁（synchronized），开销会稍微大一点，稍微有点慢。
💡 总结：两步看懂单例
单例模式说白了就是两句话：
1. 外面不准 new（构造方法 private）。
2. 统一从我这拿（提供一个 public static 的 getInstance() 方法）。

你在看一些成熟的代码时，只要看到某个类的构造方法是 private，并且里面有一个 getInstance() 的方法，不用怀疑，它百分之百就是单例模式。

⑤ 专业术语归纳

维度 说明
设计意图 保证一个类仅有一个实例，并提供一个全局访问点（Global Access Point）
所属分类 GoF 23 种设计模式中的创建型模式（Creational Pattern）
核心约束 ① 构造方法 private（禁止外部实例化）
② 内部持有唯一实例（private static 字段）
③ 对外暴露 public static 的访问方法
饿汉式 vs 懒汉式 饿汉式：类加载时即完成实例化（JVM 类加载机制保证线程安全，无需同步）
懒汉式：首次调用 getInstance() 时才实例化（延迟初始化 / Lazy Initialization），需 synchronized 保证线程安全
线程安全 饿汉式天然线程安全（static final 由 JVM 保证只初始化一次）；懒汉式需加锁，面试常考的进阶写法是双重检查锁（DCL, Double-Checked Locking）——外层 if 避免不必要的锁竞争，内层 if 防止重复创建
典型应用场景 数据库连接池（DataSource）、线程池（ThreadPoolExecutor）、全局配置（Configuration）、日志工厂（LoggerFactory）、Spring 容器中的默认 Bean 作用域（Singleton Scope）
优点 ① 严格控制实例数量，节省内存
② 提供统一的全局访问点
③ 可以灵活扩展为固定数量的实例池（如双重锁的变体）
缺点 ① 违反单一职责原则（既管业务又管生命周期）
② 单元测试困难（全局状态难以隔离和 mock）
③ 滥用会导致隐式耦合，代码可读性下降
⚠️ 面试高频追问：DCL 为什么要加 volatile？
instance = new Singleton() 在 JVM 中并非原子操作，实际分为三步：① 分配内存 → ② 初始化对象 → ③ 引用赋值。JVM 可能对 ②③ 进行指令重排序，导致其他线程拿到一个"已分配内存但尚未初始化"的半成品对象。加上 volatile 可以禁止重排序，从根本上杜绝这个问题。

7.17 Object 类(所有类的祖先)

java.lang.Object 是 Java 类层级的根,所有类(包括数组)直接或间接继承自它。下面是高频要掌握的方法。

方法 作用 默认实现
toString() 返回对象的字符串表示 类名@十六进制 hashCode
equals(Object) 判断两个对象是否"相等" 等同于 ==(比较地址)
hashCode() 返回对象的哈希值 JVM 实现,通常基于地址
getClass() 返回对象的运行时类 —
clone() 对象浅拷贝 需实现 Cloneable 接口
finalize() GC 前回调(JDK 9 起已弃用) 空实现
wait/notify 线程同步原语 需在 synchronized 块中调用

① toString() 重写

public class Point {
    int x, y;
    @Override
    public String toString() {
        return "Point(" + x + "," + y + ")";
    }
}
System.out.println(new Point());   // 自动调用 toString

② equals() 与 hashCode() 同时重写

契约:两个对象 equals 返回 true,则 hashCode 必须相同(反之不必)。否则在 HashMap、HashSet 中行为异常。

public class User {
    private String id;
    private String name;

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof User u)) return false;
        return Objects.equals(id, u.id);
    }

    @Override
    public int hashCode() {
        return Objects.hash(id);
    }
}
⚠️ 常见坑:只重写 equals,不重写 hashCode,放入 HashSet 后查不到。IDE 一般能自动生成这两个方法,务必成对生成。

7.18 枚举类(enum)

枚举用于表示一组固定的常量,如星期、性别、订单状态。JDK 5+ 引入,本质是一个继承自 java.lang.Enum 的特殊类。

① 基本定义

public enum Season {
    SPRING, SUMMER, AUTUMN, WINTER;   // 4 个常量,都是 Season 实例
}

Season s = Season.SPRING;
switch (s) {
    case SPRING -> System.out.println("春");
    case SUMMER -> System.out.println("夏");
    default    -> System.out.println("秋冬");
}

② 带字段、方法的枚举

public enum Status {
    SUCCESS(200, "成功"),
    NOT_FOUND(404, "未找到"),
    SERVER_ERROR(500, "服务器错误");

    private final int code;
    private final String msg;

    Status(int code, String msg) {   // 构造方法必须是 private(默认即是)
        this.code = code;
        this.msg = msg;
    }
    public int getCode() { return code; }
    public String getMsg() { return msg; }
}

System.out.println(Status.NOT_FOUND.getCode());   // 404

③ 枚举的常用方法

方法 说明
values() 返回所有枚举常量数组
valueOf(String) 根据名称返回对应枚举
name() 返回常量名(字符串)
ordinal() 返回常量声明顺序(从 0 开始) 
for (Season s : Season.values()) {
    System.out.println(s.name() + " " + s.ordinal());
}

Season s = Season.valueOf("SPRING");

④ 枚举实现接口

interface Describable { String describe(); }

public enum Color implements Describable {
    RED    { public String describe() { return "火热"; } },
    BLUE   { public String describe() { return "冷静"; } };
}
💡 推荐:表示"固定集合的常量"优先使用 enum 而不是 public static final int。enum 拥有类型安全、可读性、可附加行为等全面优势,还能在 switch 中使用。

7.19 记录类 Record(JDK 14+ 预览,16+ 正式)

Record 是 Java 提供的"数据载体"语法糖,用一行声明就能生成不可变的 DTO 类,自动包含字段、构造器、accessor、equals、hashCode、toString。

① 基本语法

// 一行定义
public record Point(int x, int y) { }

// 等价于传统的几十行代码:
// public final class Point {
//     private final int x;
//     private final int y;
//     public Point(int x, int y) { ... }
//     public int x() { return x; }
//     public int y() { return y; }
//     // equals / hashCode / toString 全部自动生成
// }

Point p = new Point(3, 4);
System.out.println(p.x());    // 3, 注意 accessor 是 x() 不是 getX()
System.out.println(p);        // Point[x=3, y=4]

② 紧凑构造方法、增加方法

public record Range(int low, int high) {
    // 紧凑构造:做参数校验
    public Range {
        if (low > high) throw new IllegalArgumentException("low > high");
    }
    // 可以增加普通方法
    public int length() { return high - low; }
}

③ Record 的限制

  • 隐式 final,不能被继承
  • 不能 extends 任何类(隐式继承 java.lang.Record),但可以实现接口
  • 不能定义实例字段(只能在头部声明),所有字段隐式 final
  • 仅适合做不可变的数据载体(DTO、值对象),不适合做实体类(JPA 实体)
💡 实战:DTO、VO、配置对象、Result<T> 返回值 —— 这些都是 Record 的最佳应用场景。简单几行就能替代过去用 Lombok @Value 生成的代码。
