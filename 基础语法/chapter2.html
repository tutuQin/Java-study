第二章 变量与常量

变量是程序中存储数据的"容器",其本质是内存中的一块命名区域。Java 是强类型语言,任何变量在使用前都必须先声明类型。

2.1 变量的定义

声明变量需要三个要素:数据类型、变量名、初始值(可选)。

// 语法格式: 数据类型 变量名 = 值;
int age = 18;                // 声明同时初始化
String name;                // 先声明
name = "Tom";                // 后赋值

// 一行声明多个同类型变量(不推荐,可读性差)
int a = 1, b = 2, c = 3;

// JDK 10+ 局部变量类型推断 var
var message = "Hello";      // 自动推断为 String
var list = new ArrayList<Integer>(); // 推断为 ArrayList<Integer>
📌 注意:
  • 变量必须先声明、后使用
  • 同一作用域内,变量名不能重复
  • 局部变量必须先初始化才能使用
  • var 只能用于局部变量,且声明时必须初始化

2.2 变量分类与作用域

按照变量在类中的位置,可分为三类:

变量类型声明位置作用域默认值存储位置
成员变量(实例)类中、方法外整个类,跟随对象有堆
静态变量(类变量)类中、加 static整个类,所有对象共享有方法区/元空间
局部变量方法/代码块内所属代码块无,必须手动初始化栈
 
public class Student {
    private String name;            // 成员变量 (实例变量)
    private static int count;      // 静态变量 (类变量)

    public void study() {
        int hours = 2;             // 局部变量
        if (hours > 0) {
            String book = "Java";    // 块级局部变量,仅在 if 内有效
            System.out.println(book);
        }
        // System.out.println(book); ❌ 此处无法访问 book
    }
}
💡 作用域规则:变量的作用域从声明处开始,到包含它的最近一对大括号 {} 结束。

2.3 命名规范

① 硬性规则(必须遵守,否则编译报错)

  • 由字母、数字、下划线 _、美元符 $组成
  • 不能以数字开头
  • 不能使用 Java 关键字和保留字(如 int、class、true)
  • 严格区分大小写(age 与 Age 不同)
  • 长度无限制,但建议简洁明了

② 软性规范(行业约定,提高可读性)

类型命名规则示例
类名 / 接口名大驼峰(UpperCamelCase)HelloWorld, UserService
变量名 / 方法名小驼峰(lowerCamelCase)userName, getCount()
常量名全大写 + 下划线分隔MAX_SIZE, PI
包名全小写,以 . 分隔com.example.demo
 
// ✅ 推荐
int studentAge = 18;
final double PI = 3.14159;
public class UserService { }

// ❌ 不推荐
int a = 18;              // 命名无意义
int student_age = 18;    // 应使用小驼峰
int 2age = 18;            // ❌ 不能数字开头
int class = 1;           // ❌ 关键字

2.4 常量(final)

常量是值不可改变的标识符,使用 final 关键字声明。一旦赋值,后续不能再修改。

// ① 局部常量
final int MAX = 100;
// MAX = 200; ❌ 编译错误: cannot assign a value to final variable

// ② 成员常量(实例常量)
public class Circle {
    public final double radius;
    public Circle(double r) {
        this.radius = r;          // 可在构造方法中赋值
    }
}

// ③ 静态常量(全局常量)—— 最常用
public class MathUtil {
    public static final double PI = 3.14159265358979;
    public static final int MAX_USERS = 1000;
}
// 使用: MathUtil.PI

final 关键字的作用

修饰对象含义
修饰变量变成常量,只能赋值一次
修饰方法不能被子类重写(override)
修饰类不能被继承,如 String、Integer
⚠️ 注意:对引用类型来说,final 只保证引用不变,不保证对象内容不变。 
final List<Integer> list = new ArrayList<>();
list.add(1);                // ✅ 可以,内容可改
// list = new ArrayList<>(); ❌ 引用不可改

2.5 关键字与保留字

Java 共有 50 个关键字 + 2 个保留字(goto、const,目前未使用但禁止作为标识符)。三个特殊字面量 true、false、null 严格意义上不是关键字,但同样不能作为标识符。

分类关键字
数据类型byte short int long float double char boolean void
流程控制if else switch case default for while do break continue return
类与对象class interface enum extends implements new this super instanceof
修饰符public protected private static final abstract synchronized volatile transient native strictfp
异常处理try catch finally throw throws
包相关package import
其他assert (JDK 1.4+), yield (JDK 14+)
保留字goto const
特殊字面量true false null
📌 提示:JDK 10+ 的 var 和 JDK 14+ 的 record、sealed 是上下文关键字(reserved type names),在特定位置才有特殊含义,允许作为普通标识符使用(但不推荐)。
